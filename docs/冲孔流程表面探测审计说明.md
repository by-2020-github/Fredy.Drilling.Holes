# 冲孔流程表面探测审计说明

## 1. 审计范围

本文只审计自动冲孔流程中的“表面探测”相关行为，重点回答以下问题：

- 自动冲孔在什么时机探测表面。
- 探测参数从哪里来，默认值和兜底逻辑是什么。
- 探测结果如何进入后续冲孔补偿。
- 哪些失败现象是参数问题，哪些更像底层运动问题。

本说明不展开校准页测试冲孔的页面交互，只在涉及全局参考 Z 来源时做必要说明。

## 2. 运行时调用链

自动冲孔表面探测的实际调用链如下：

```text
AppConfig
  -> MainViewModel.CreatePunchStateMachine()
  -> PunchStateMachine
  -> IHardwareController / Adt8940Controller
  -> SurfaceDetectionService
  -> Motion / IO / Lock 硬件
```

各层职责分工：

- `MainViewModel` 负责把全局配置映射到 `PunchStateMachine`。
- `PunchStateMachine` 决定“何时探、探哪里、探完后怎么参与补偿”。
- `Adt8940Controller` 负责把“预探”和“正式冲孔中的表面记录”落到真实运动命令。
- `SurfaceDetectionService` 负责执行统一的探面算法，支持 `Latch` 和 `IoPolling` 两种模式。

## 3. 自动冲孔中的实际探测流程

### 3.1 启动时先装载全局参考 Z

状态机创建时，会先从全局配置读取：

- `HasWorkpieceReferenceZ`
- `WorkpieceReferenceZ`

这组值在流程启动时被装载为初始参考面：

```text
_hasSurfaceReference = HasInitialSurfaceReference
_referenceSurfaceZ = InitialSurfaceReferenceZ
_latestSurfaceZ = InitialSurfaceReferenceZ
```

这意味着：

- 如果当前工件已有历史参考 Z，即使尚未开始首孔预探，状态机也已经具备一个初始补偿基准。
- 但只要自动冲孔启用了表面探测，首孔左侧预探成功后，新的探测值会覆盖这个初始参考面。

### 3.2 首孔左侧预探只发生在第一个孔

进入 `DetectSurface` 状态后，只有满足以下条件才会做预探：

- `IsDetectionEnabled == true`
- `CurrentHoleIndex == 1`

也就是说：

- 非首孔不会再次做“左侧测试点慢速探测”。
- 后续孔位默认依赖已建立的表面参考和样本补偿。

预探过程分两步：

1. XY 先移动到第一个正式孔的左侧偏移点。
2. 调用 `Hardware.ProbeSurface(...)` 进行 Z 轴探面。

默认偏移语义：

```text
probeX = firstHoleX + SurfaceProbeOffsetX
probeY = firstHoleY + SurfaceProbeOffsetY
```

当前默认值是：

- `SurfaceProbeOffsetX = -1`
- `SurfaceProbeOffsetY = 0`

### 3.3 首孔左侧预探成功后会更新基准面

首孔左侧预探成功后，状态机会立即执行：

```text
UpdateSurfaceReference(probeSurfaceZ)
_hasBaselineSurfaceProbe = true
```

结果是：

- `_referenceSurfaceZ` 被更新为本轮工件实际测到的首个基准面。
- `_latestSurfaceZ` 同步更新为该值。
- 当前这次预探结果不会直接作为“最近邻样本点”写入 `_surfaceSamples`。

这个区别很重要：

- 左侧预探点只负责建立整轮工件的基准面。
- 它不是正式孔位，因此不参与最近邻欧式距离匹配。

### 3.4 首个正式孔的首刀，可能再次记录当前点 Surface Z

首个正式孔开始冲孔时，状态机会判断是否需要“在正式下压过程中记录当前孔位的表面 Z”。

触发条件是：

```text
CurrentHoleIndex == 1
&& !_hasFirstPunchSurfaceSample
&& _hasBaselineSurfaceProbe
```

因此只有下面这种场景会发生：

- 首孔左侧预探已经成功。
- 当前正在执行第一个正式孔。
- 这是第一次记录正式孔位样本。

如果命中探测信号：

- 记录当前正式孔位的 `SurfaceZ`。
- 将该点写入 `_surfaceSamples`。
- 后续孔位开始使用“最近邻样本”补偿。

如果没有命中探测信号：

- 不会中断本次冲孔。
- 状态机会继续沿用首孔左侧预探得到的参考面。
- `_hasFirstPunchSurfaceSample` 仍会被置为 `true`，因此不会再次尝试为首孔补录表面样本。

### 3.5 后续孔位不再主动探面，而是走最近邻补偿

从第二个孔开始，`DetectSurface` 状态直接跳过。

后续补偿规则：

1. 若 `_surfaceSamples` 为空，则补偿量使用：

```text
compensation = _latestSurfaceZ - _referenceSurfaceZ
```

在常见场景下，这通常等于 `0`，因为首孔左侧预探后 `_latestSurfaceZ` 与 `_referenceSurfaceZ` 相同。

2. 若 `_surfaceSamples` 非空，则从样本集合中按 XY 欧式距离选择最近点：

```text
compensation = nearestSample.SurfaceZ - _referenceSurfaceZ
```

3. 本孔实际使用的局部表面 Z 为：

```text
localSurfaceZ = _referenceSurfaceZ + compensation
```

等价于：

```text
localSurfaceZ = nearestSample.SurfaceZ
```

### 3.6 冲孔目标 Z 的计算方式

当已经建立表面参考后，自动冲孔按绝对 Z 目标执行：

```text
targetZ = localSurfaceZ - configuredDepth
```

其中：

- Z 轴向下为负方向。
- `configuredDepth` 使用配方中的正值深度量。

如果尚未建立任何表面参考，则状态机会退化为相对下压：

```text
fallbackRelativeDepth = -stepDepth
```

这属于兜底路径，不是优先工艺路径。

### 3.7 每刀冲孔后都会回到全局 SafeZ

无论头道还是二道，只要该刀冲孔执行完成，状态机会做以下动作：

```text
WaitForZStop()
LiftZ(PunchSafeZ, FastToSafeZSpeed)
```

这表示：

- 每一刀都先确认 Z 轴停稳。
- 然后回到自动冲孔专用的全局绝对安全位 `PunchSafeZ`。
- 头道多段冲孔之间也会回 SafeZ，而不是在低位连续累压。

## 4. SurfaceDetectionService 的实际探面算法

`SurfaceDetectionService` 的标准探面算法如下：

1. 探测前准备。
2. 先执行快速接近段。
3. 快速段完成后立即读一次探测结果。
4. 若快速段已命中，直接抛异常。
5. 重新准备探测状态。
6. 再执行慢速搜索段。
7. 慢速运动过程中持续轮询探测结果。
8. 一旦命中，立即停 Z，并等待 Z 完全停止。
9. 如果整段慢速运动结束仍未命中，再做一次最终读取。

这里最关键的审计结论是：

- 快速段的设计目标是“快速靠近表面，但不能提前触发表面信号”。
- 真正允许触发表面信号的，是后续慢速搜索段。
- 如果快速段已经触发，系统会认为当前针高、工件高度或传感器状态异常。

## 5. 预探与正式冲孔内探测的行为差异

虽然两者都复用了同一套探测模式语义，但运行行为并不完全一样。

### 5.1 首孔左侧预探

调用入口：`Hardware.ProbeSurface(...)`

行为特点：

- 快速段后会做提前命中检查。
- 慢速段命中后会立即停 Z。
- 返回值直接决定“首孔左侧预探是否成功”。
- 未检测到表面时，自动冲孔会报警并结束流程。

### 5.2 正式冲孔中的表面记录

调用入口：`Hardware.PunchDown(..., detectSurface: true, ...)`

行为特点：

- 仅在首个正式孔首刀且已完成左侧预探时才会启用。
- 下压动作会直接向最终冲孔目标 Z 运动。
- 运动过程中只要检测到表面，就记录一次 `SurfaceZ`。
- 记录后不会像预探那样立即停 Z，而是继续完成本次冲孔动作。

这个差异是符合现有工艺语义的：

- 预探的目标是“找到表面”。
- 正式冲孔中的检测目标是“在正常冲孔过程中顺手采样当前孔位表面”，不是重新做一次停机式探面。

## 6. 参数来源审计

### 6.1 参数入口

自动冲孔探面相关参数目前统一来自全局配置页的“冲孔表面探测”页签。

### 6.2 参数映射表

| 配置字段 | 默认值 / 归一化 | 映射到状态机 | 实际用途 |
| --- | --- | --- | --- |
| `HasWorkpieceReferenceZ` | 默认 `false` | `HasInitialSurfaceReference` | 是否在流程启动时装载全局参考 Z |
| `WorkpieceReferenceZ` | 默认 `0` | `InitialSurfaceReferenceZ` | 初始参考面 Z；首孔预探成功后会被新的探测值覆盖 |
| `FastMovePos` | 默认 `-22.0` | `FastApproachDistance` | 预探快速接近段的相对位移 |
| `FastMoveSpeed` | 默认 `9.0` | `FastApproachSpeed` | 预探快速接近段速度 |
| `SlowMoveDist` | 默认 `-12.0` | `SlowDetectDistance` | 预探慢速搜索段的相对位移 |
| `SlowMoveSpeed` | 默认 `0.7` | `SlowDetectSpeed` | 预探慢速搜索段速度 |
| `PunchSafeZ` | 默认 `8500` | `SafeZ` | 每刀冲孔结束后的全局绝对回退 Z |
| `FastToSafeZSpeed` | 小于 `0` 时归一化为 `0` | `FastToSafeZSpeed` | 回 SafeZ 速度；`0` 表示沿用 Z 轴默认速度 |
| `PunchDownSpeed` | 小于 `0` 时归一化为 `0` | `PunchDownSpeed` | 正式冲孔下压速度；`0` 表示沿用 Z 轴默认速度 |
| `SurfaceProbeOffsetX` | 默认 `-1` | `ReferenceProbeOffsetX` | 首孔左侧预探 X 偏移 |
| `SurfaceProbeOffsetY` | 默认 `0` | `ReferenceProbeOffsetY` | 首孔左侧预探 Y 偏移 |
| `SurfaceDetectionMode` | 为空时归一化为 `Latch` | `SurfaceDetectionOptions.Mode` | 探测模式，`Latch` 或 `IoPolling` |
| `SurfaceDetectInputPort` | 默认 `0` | `SurfaceDetectionOptions.InputPort` | `IoPolling` 模式下的输入口 |
| `SurfaceDetectInputLowActive` | 默认 `true` | `SurfaceDetectionOptions.InputLowActive` | `IoPolling` 模式下的有效电平 |
| `SurfaceDetectPollIntervalMs` | 小于等于 `0` 时归一化为 `10` | `SurfaceDetectionOptions.PollIntervalMs` | 探测轮询间隔 |

### 6.3 两个容易混淆的参数边界

1. `PunchSafeZ` 只属于自动冲孔流程。

校准页测试冲孔自己的恢复高度已拆分为页面专属配置，不再共用这个字段。

2. `WorkpieceReferenceZ` 是“启动时初始参考值”，不是“锁死不变的最终参考值”。

只要自动冲孔启用首孔预探并成功，状态机就会用本轮新测到的表面 Z 覆盖启动时装载的旧参考值。

## 7. 失败现象与排查结论

### 7.1 “在设定搜索距离内未检测到表面信号”通常是搜索窗口不够

如果当前 SafeZ、快速段和慢速段合计只覆盖到某个较浅位置，而实际工件表面更低，就会出现未探到表面的情况。

应重点检查：

- `PunchSafeZ` 当前绝对值是否正确。
- `FastMovePos + SlowMoveDist` 是否真的覆盖到实际表面所在 Z。
- 参数符号是否符合“向下为负”的约定。

例如：

```text
当前 Z = 8500
FastMovePos = -8000
SlowMoveDist = -1000
```

那么搜索最低只到：

```text
8500 - 8000 - 1000 = -500
```

如果真实表面还在更低位置，自然不会探到。

### 7.2 快速段提前触发不是“探到了”，而是异常

若系统报出“快速接近阶段已触发表面检测信号”，应优先怀疑：

- 针尖起始高度过低。
- 工件或夹具高度异常。
- 传感器输入已处于触发态。
- 锁存状态未正确清除。

这不是慢速段参数太小的问题，而是快速段已经失去了“只靠近不接触”的工艺前提。

### 7.3 `Latch` 模式不是通用模式

`SurfaceDetectionService` 明确要求：

- `Latch` 模式只能在 ADT8940 控制器下使用。

如果当前控制器不支持 ADT 锁存，系统会直接抛异常，必须改为 `IoPolling`。

### 7.4 `IoPolling` 模式的精度与速度、轮询间隔强相关

在 `IoPolling` 模式下，记录到的 `SurfaceZ` 不是硬件锁存瞬间位置，而是“软件读到输入有效时，当前轴位置”。

因此以下参数会共同影响结果：

- `SurfaceDetectPollIntervalMs`
- `SlowMoveSpeed`
- `PunchDownSpeed`

速度越快、轮询越慢，记录到的 `SurfaceZ` 可能越滞后。

### 7.5 若日志显示已停机，但实际位置与目标严重不符，问题不一定在探面参数

此前现场现象显示，存在“运动层已经认为 Z 停止，但实际位置与命令位置仍差异较大”的可疑情况。

如果再次出现类似现象，应优先把问题切分为两层：

1. 探面层是否真的已经给出正确的目标 Z。
2. 运动层是否可靠地把 Z 轴执行到目标位置，并且其停止判定是否可信。

这类问题更接近 `MotionManager` / `HAL` / 驱动反馈一致性，不应只通过调大调小探面参数来掩盖。

## 8. 当前实现的结论性摘要

当前自动冲孔的表面探测语义可以概括为：

1. 启动时先装载全局参考 Z 作为初始基准。
2. 若启用探面，则只在首孔左侧做一次正式预探，并用结果覆盖当前基准面。
3. 首个正式孔首刀在正常冲孔过程中尝试补录当前点 `SurfaceZ`。
4. 后续孔位不再逐孔探面，而是按最近邻样本做补偿。
5. 每刀完成后都回到全局 `PunchSafeZ`。

这套逻辑的关键前提有两个：

- 首孔左侧预探必须真实、稳定地建立本轮基准面。
- Z 轴底层运动状态必须可信，否则补偿计算正确也可能落不到真实位置。

## 9. 建议的现场核对顺序

1. 先确认 `PunchSafeZ`、`FastMovePos`、`SlowMoveDist` 的组合是否覆盖真实表面高度。
2. 再确认 `SurfaceDetectionMode`、输入口号、电平极性是否与现场接线一致。
3. 若使用 `IoPolling`，先把 `SlowMoveSpeed` 和 `PunchDownSpeed` 降下来，再看 `SurfaceZ` 是否稳定。
4. 若仍出现“停了但位置不对”，转向排查运动层反馈，不要继续只改探面参数。
