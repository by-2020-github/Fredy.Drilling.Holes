# 自动冲孔表面探测与补偿工艺说明

## 1. 坐标与核心约定

### 1.1 Z 轴方向

机台坐标系约定：**Z 轴向下为负方向**。

因此配方中的冲孔深度使用正值表示“下压深度量”，实际冲孔目标 Z 需要换算为更小的绝对 Z 坐标：

```text
targetZ = surfaceZ - punchDepth
```

### 1.2 全局安全位 safeZ

所有工件类型共用一个全局 `safeZ`。

- `safeZ` 是 **绝对坐标**。
- 自动冲孔中所有回安全位动作都应直接移动到该绝对 Z。
- 当前配置字段为：`AppConfig.PunchSafeZ`。
- 回 `safeZ` 的速度由 `AppConfig.FastToSafeZSpeed` 控制；当该值为 `0` 时沿用 Z 轴参数速度。
- 相机冲孔偏移校准页中的“安全 Z”会同步保存到全局 `PunchSafeZ`。

## 2. 开始冲孔前的表面基准探测

由于所有工件表面都存在翘曲，自动冲孔正式开始前，需要先建立本轮工件的表面基准。

### 2.1 探测位置

在第一个孔的左侧取一个测试点进行表面探测。

默认偏移：

```text
X = 第一个孔 X - 1mm
Y = 第一个孔 Y
```

对应配置字段：

- `AppConfig.SurfaceProbeOffsetX`，默认 `-1`
- `AppConfig.SurfaceProbeOffsetY`，默认 `0`

### 2.2 探测目的

该测试点探测得到的 Z 值作为本轮冲孔流程的基准表面 Z：

```text
referenceSurfaceZ = probeSurfaceZ
```

注意：

- 该左侧测试点只用于建立基准。
- 它不作为正式孔位的最近邻 `SurfaceZ` 样本。
- 第一个正式孔仍需要在首刀冲孔过程中记录自己的 `SurfaceZ`。

## 3. 正式冲孔表面 Z 记录规则

### 3.1 第一个正式孔

第一个正式孔的 Z 补偿先使用左侧测试点得到的 `referenceSurfaceZ`。

在第一个正式孔的慢速/多次冲孔过程中：

- **只有第一次冲孔会探测表面**。
- 探测到的 Z 记录为该正式孔的 `SurfaceZ`。
- 该 `SurfaceZ` 会加入样本集合，用于后续孔位补偿。

### 3.2 第二个及之后的孔

从第二个孔开始，不再每孔主动探测表面。

每个孔根据 XY 欧式距离寻找最近的已记录 `SurfaceZ` 样本：

```text
nearestSample = min distance((holeX, holeY), (sampleX, sampleY))
```

该孔的补偿计算为：

```text
compensation = nearestSample.SurfaceZ - referenceSurfaceZ
localSurfaceZ = referenceSurfaceZ + compensation
targetZ = localSurfaceZ - configuredDepth
```

由于 `localSurfaceZ = nearestSample.SurfaceZ`，实际等价于：

```text
targetZ = nearestSample.SurfaceZ - configuredDepth
```

## 4. 多次冲孔深度规则

### 4.1 头道冲孔

头道冲孔支持多段深度。

配方中的每段深度表示本次新增的下压深度量，执行时按累计深度计算目标 Z。

示例：

```text
No.1 = 0.04
No.2 = 0.04
No.3 = 0.02
```

执行目标深度为：

```text
第 1 次：0.04
第 2 次：0.04 + 0.04 = 0.08
第 3 次：0.04 + 0.04 + 0.02 = 0.10
```

对应目标 Z：

```text
targetZ1 = localSurfaceZ - 0.04
targetZ2 = localSurfaceZ - 0.08
targetZ3 = localSurfaceZ - 0.10
```

### 4.2 二道冲孔

二道冲孔当前使用独立的 `SecondPunchDepth`。

```text
targetZ = localSurfaceZ - SecondPunchDepth
```

## 5. 每次冲孔后的安全回退

工艺要求：每次冲孔完成后都回到全局绝对 `safeZ`。

当前执行语义：

```text
PunchDown(targetZ)
WaitForZStop()
LiftZ(safeZ)
```

其中：

- 头道和二道正式冲孔下压速度统一使用 `AppConfig.PunchDownSpeed`。
- 回 `safeZ` 速度使用 `AppConfig.FastToSafeZSpeed`。
- 两个速度配置为 `0` 时，均沿用 Z 轴参数中的速度。

这保证多次啄钻式冲孔过程中，每一刀完成后都会先回到安全 Z，再进行下一刀。

## 6. 表面 Z 探测方式

表面 Z 探测支持两种方式。

配置字段：

- `AppConfig.SurfaceDetectionMode`
  - `Latch`
  - `IoPolling`
- `AppConfig.SurfaceDetectInputPort`
- `AppConfig.SurfaceDetectInputLowActive`
- `AppConfig.SurfaceDetectPollIntervalMs`

### 6.1 锁存方式 Latch

锁存方式依赖运动控制卡的锁存状态和锁存位置。

流程：

```text
ClearLockStatus
SetLockPositionMode
Z 轴向下运动
检测 LockStatus
读取 LockPosition 作为 SurfaceZ
```

适用场景：

- 控制卡锁存功能稳定。
- 对接触瞬间位置精度要求较高。

### 6.2 IO 轮询方式 IoPolling

IO 方式仿照相机冲孔偏移校准页的测试冲孔逻辑。

流程：

```text
Z 轴慢速向下运动
循环读取指定输入 IO
IO 触发时读取当前 Z
记录该 Z 作为 SurfaceZ
```

区别：

- 开工前左侧测试点预探时，检测到 IO 后会停止 Z。
- 正式冲孔过程中记录首个正式孔 `SurfaceZ` 时，检测到 IO 后只记录 Z，不停止本次冲孔。

## 7. 自动冲孔整体流程

```text
1. 读取配方和孔位
2. 读取全局 safeZ
3. 移动到第一个孔
4. 移动到第一个孔左侧 1mm 测试点
5. 按配置的探测方式探测表面，得到 referenceSurfaceZ
6. 回到 safeZ
7. 回到第一个正式孔
8. 第一个孔第一刀冲孔，同时记录当前点 SurfaceZ
9. 每一刀冲孔后回 safeZ
10. 后续孔根据最近 SurfaceZ 样本进行补偿
11. 全部孔完成后保持 safeZ
```

## 9. 后续建议

1. 参数配置界面已补充“冲孔表面探测”页签，可视化编辑以下字段：
   - `PunchSafeZ`
   - `FastToSafeZSpeed`
   - `PunchDownSpeed`
   - `SurfaceDetectionMode`
   - `SurfaceProbeOffsetX`
   - `SurfaceProbeOffsetY`
   - `SurfaceDetectInputPort`
   - `SurfaceDetectInputLowActive`
   - `SurfaceDetectPollIntervalMs`
2. 实机验证时优先确认：
   - `safeZ` 绝对坐标是否安全。
   - 左侧 1mm 测试点是否避开孔位且仍在工件表面。
   - 锁存模式和 IO 模式记录的 `SurfaceZ` 是否一致。
   - 第一正式孔首刀记录的 `SurfaceZ` 是否进入后续最近邻补偿。
