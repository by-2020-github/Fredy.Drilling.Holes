# ProcessManager.cs 与 MotionAdt8940.cs 回零方式对比分析报告

## 1. 分析目标

对比以下两处回零实现，重点确认“光栅尺回零参数/端口参数”是否存在，以及两者实现方式是否一致：

- `Fredy\Windows\ParamConfig\old\ProcessManager.cs`
  - `HomeXYAxis`
  - `HomeZAxis`
- `HAL\Motion\MotionAdt8940.cs`
  - `HomeXYAxisAsync`
  - `HomeZAxisAsync`

---

## 2. 结论先说

### 2.1 MotionAdt8940 不是“没有端口参数”，而是改成了配置注入模式

`MotionAdt8940.cs` 中的 `HomeXYAxisAsync` / `HomeZAxisAsync` 方法签名里确实**没有直接出现端口参数**，但它们并不是没有端口配置。

它把回零相关端口、有效电平、超时、回退脉冲、抬升脉冲、光栅模式等，统一封装在：

- `HomingPort`
- `HomingOptions`

然后通过：

- `ConfigureHoming(HomingOptions options)`

提前注入到 `_homingOptions` 字段中，回零时再从 `_homingOptions` 读取。

也就是说：

- `ProcessManager`：参数是**直接从 `GlobalParams.Instance` 临时读取**
- `MotionAdt8940`：参数是**先组装成 `HomingOptions`，再在内部使用**

---

## 3. 关键代码定位

## 3.1 ProcessManager 中的回零参数来源

### XY 轴机械零位参数
`ProcessManager.cs` 中 `HomeXYAxis`：

- 机械零位端口：
  - X 轴：`GlobalParams.Instance.XSTOP0`
  - Y 轴：`GlobalParams.Instance.YSTOP0`
- 机械零位极性：
  - X 轴：`GlobalParams.Instance.check_XSTOP0`
  - Y 轴：`GlobalParams.Instance.check_YSTOP0`

### XY 轴光栅尺回零参数
同一方法中明确存在：

- `bool ioMode = visonCAM.GlobalParams.Instance.Home_IO;`
- `bool latchMode = visonCAM.GlobalParams.Instance.Home_latch;`
- `bool encoderHomeSuccess = false;`

并使用：

- X 光栅端口：`GlobalParams.Instance.XSTOPrule`
- Y 光栅端口：`GlobalParams.Instance.YSTOPrule`

### Z 轴参数
`HomeZAxis` 中只有：

- 机械零位端口：`GlobalParams.Instance.ZSTOP0`
- 机械零位极性：`GlobalParams.Instance.check_ZSTOP0`
- 回零后抬升脉冲：`GlobalParams.Instance.HomeZLocation`

**注意：`ProcessManager.HomeZAxis` 里并没有光栅尺回零步骤，也没有 `Home_IO / Home_latch / encoderHomeSuccess` 这组参数。**

所以从现有代码看：

- `HomeXYAxis` 有“机械回零 + 可选光栅尺回零”
- `HomeZAxis` 只有“机械回零 + 抬升”

---

## 3.2 MotionAdt8940 中的回零参数来源

### 配置结构
`MotionAdt8940.cs` 中定义了：

### `HomingPort`
用于表示一个回零端口：

- `PortIndex`
- `IsLowLevelActive`

### `HomingOptions`
包含：

- `IsIoHome`
- `IsLatch`
- `IsGratingHome`
- `XLimitPort`
- `YLimitPort`
- `ZLimitPort`
- `XGratingPort`
- `YGratingPort`
- `HomeTimeoutMs`
- `HomeBackoffPulse`
- `ZHomeLiftPulse`
- `ZHomeTowardPositiveDirection`
- 慢速回零速度参数
- 光栅回零速度参数

### 注入方式
`ConfigureHoming(HomingOptions options)` 把配置保存到 `_homingOptions`。

调用链上，这些参数来自：

- `Fredy\App.xaml.cs` 中的 `BuildAdtHomingOptions`
- `Fredy\Windows\ParamConfig\ConfigViewModel.cs` 中的 `BuildAdtHomingOptions`
- 配置模型：`ConfigModels.cs`
  - `AppConfig`
  - `AdtHomingConfig`
  - `PortItem`

所以 `MotionAdt8940` 的端口不是没有，而是：

- 机械限位端口：`_homingOptions.XLimitPort / YLimitPort / ZLimitPort`
- 光栅端口：`_homingOptions.XGratingPort / YGratingPort`
- 极性：`PortItem.IsLowLevelActive`

---

## 4. XY 回零流程对比

## 4.1 ProcessManager.HomeXYAxis

流程大致如下：

1. 读取机械零位端口和电平配置
2. 根据当前输入电平决定先走哪个方向
3. 先做一次较快搜索
4. 退回 `200` 脉冲（仅某一分支显式做）
5. 再慢速逼近机械零位
6. `Setup_Pos(axis, 0, 0)` 置零
7. 若 `Home_rule == true`，再执行光栅尺回零：
   - IO 模式：监测 `XSTOPrule / YSTOPrule`
   - 锁存模式：`Setup_LockPosition` + `Get_LockStatus` + `Get_LockPosition`
8. 成功后置回零标志位

特点：

- 参数来源于 `GlobalParams.Instance`
- 代码是阻塞式 `while + Thread.Sleep(10)`
- 带急停检查 `FlagEmergencyStop`
- 重试次数固定 3 次
- 光栅尺回零模式由：
  - `Home_rule`
  - `Home_IO`
  - `Home_latch`
  控制

---

## 4.2 MotionAdt8940.HomeXYAxisAsync

流程大致如下：

1. 调用 `HomeXYMechanicalAsync`
2. 若 `_homingOptions.IsGratingHome == true`，再进入光栅尺回零：
   - `IsLatch == true`：`HomeXYByLatchAsync`
   - `IsIoHome == true`：`HomeXYByGratingIoAsync`
3. 最后 `ResetPosition(axisNo)` 清零

其中：

### 机械回零 `HomeXYMechanicalAsync`
- 从 `_homingOptions.XLimitPort / YLimitPort` 取机械零位端口
- 通过 `GetActiveLevel / GetInactiveLevel` 自动计算有效/无效电平
- 先快速搜索
- 再按 `HomeBackoffPulse` 回退
- 再低速逼近

### 光栅 IO 回零 `HomeXYByGratingIoAsync`
- 从 `_homingOptions.XGratingPort / YGratingPort` 取端口
- 按 `GratingHomeStartSpeed / GratingHomeSpeed / GratingHomeAcceleration` 执行连续运动
- 等待输入达到 `GetActiveLevel(port)`

### 光栅锁存回零 `HomeXYByLatchAsync`
- 清锁存状态
- 配置锁存模式
- 启动连续运动
- 等待锁存完成
- 读取锁存位置
- 移动到锁存位置
- 最后清除锁存模式和状态

特点：

- 参数统一来自 `_homingOptions`
- 是异步实现：`async/await + CancellationToken`
- 有超时机制
- 取消时走 `EmergencyStopAsync`
- 光栅回零模式对应关系为：
  - `IsGratingHome`
  - `IsIoHome`
  - `IsLatch`

---

## 5. Z 轴回零流程对比

## 5.1 ProcessManager.HomeZAxis

实现内容：

1. 读取 `ZSTOP0`
2. 根据当前电平判断回零方向
3. 连续运动直到检测到机械零位信号
4. `Setup_Pos(3, 0, 0)` 清零
5. 若 `HomeZLocation > 0`，则正向抬升对应脉冲
6. 设置 `FlaghomeZ = true`

### 关键结论

`ProcessManager.HomeZAxis` **没有实现光栅尺回零**，也没有：

- `Home_IO`
- `Home_latch`
- `encoderHomeSuccess`
- Z 光栅端口

所以如果你的预期是“Z 轴光栅尺回零也应该有这组三个参数”，那么**现有 `ProcessManager.cs` 本身并没有实现**。

---

## 5.2 MotionAdt8940.HomeZAxisAsync

实现内容：

1. 通过 `GetMechanicalPort(axisNo)` 取 `ZLimitPort`
2. 根据 `ZHomeTowardPositiveDirection` 确定搜索方向
3. 如当前已在触发态，先脱离开关
4. 再搜索到机械零位有效电平
5. `ResetPosition(axisNo)` 清零
6. 如 `ZHomeLiftPulse > 0`，则执行抬升

### 关键结论

`MotionAdt8940.HomeZAxisAsync` 同样**只有机械回零 + 抬升**，没有任何 Z 光栅回零流程。

而且在 `HomingOptions` 中只定义了：

- `XGratingPort`
- `YGratingPort`

**没有 `ZGratingPort`**。

这说明目前 `MotionAdt8940` 的设计本身就只支持：

- XY 支持机械回零后再做光栅尺回零
- Z 只支持机械回零

---

## 6. 参数映射关系对比

| 功能 | ProcessManager | MotionAdt8940 |
|---|---|---|
| XY机械零位端口 | `XSTOP0` / `YSTOP0` | `XLimitPort` / `YLimitPort` |
| Z机械零位端口 | `ZSTOP0` | `ZLimitPort` |
| XY机械零位极性 | `check_XSTOP0` / `check_YSTOP0` | `PortItem.IsLowLevelActive` |
| Z机械零位极性 | `check_ZSTOP0` | `PortItem.IsLowLevelActive` |
| 是否启用光栅回零 | `Home_rule` | `IsGratingHome` |
| 光栅 IO 模式 | `Home_IO` | `IsIoHome` |
| 光栅锁存模式 | `Home_latch` | `IsLatch` |
| X光栅端口 | `XSTOPrule` | `XGratingPort` |
| Y光栅端口 | `YSTOPrule` | `YGratingPort` |
| Z光栅端口 | 无 | 无 |

| Z回零后抬升 | `HomeZLocation` | `ZHomeLiftPulse` |
| Z回零方向 | 逻辑写死在方法里 | `ZHomeTowardPositiveDirection` |

## 7. 两者最重要的差异

## 7.1 参数组织方式不同

### ProcessManager
- 强依赖 `GlobalParams.Instance`
- 每次回零时现场读取参数
- 参数分散在多个全局字段中

### MotionAdt8940
- 参数统一封装在 `HomingOptions`
- 方法内部不直接依赖 UI/全局配置类
- 更利于测试、复用、迁移

这也是你看到“方法没有端口参数”的原因：

**不是没有，而是改成了面向配置对象的封装。**

---

## 7.2 XY 的光栅回零两边都有，Z 两边都没有

这是本次分析里最关键的结论。

### ProcessManager
- `HomeXYAxis`：有光栅尺回零
- `HomeZAxis`：没有光栅尺回零

### MotionAdt8940
- `HomeXYAxisAsync`：有光栅尺回零
- `HomeZAxisAsync`：没有光栅尺回零

所以如果你现在怀疑：

> “ProcessManager.cs 中 HomeXYAxis、HomeZAxis 的光栅尺回零应该有 `ioMode/latchMode/encoderHomeSuccess` 参数，但 MotionAdt8940.cs 中 HomeXYAxisAsync、HomeZAxisAsync 貌似没有端口参数。”

更准确的说法应该是：

1. `HomeXYAxis` 的确有这组三个局部参数
2. `HomeXYAxisAsync` 没有同名局部变量，但有等价配置：
   - `IsIoHome`
   - `IsLatch`
   - `IsGratingHome`
   - `XGratingPort / YGratingPort`
3. `HomeZAxis` 本身就没有这组三个参数
4. `HomeZAxisAsync` 也同样没有，因为当前设计压根没有实现 Z 光栅回零

---

## 8. 可能存在的迁移/行为差异点

## 8.1 ProcessManager 的 Z 轴极性变量 `a` 基本未真正参与判断

在 `HomeZAxis` 中：

- 读取了 `check_ZSTOP0`
- 计算了 `a = isCheckStop0 ? 0 : 1`

但后续判断逻辑基本都在等待电平变为 `1`，`a` 没有像 XY 那样真正参与分支判定。

这说明：

- `ProcessManager.HomeZAxis` 的极性适配并不完整
- `MotionAdt8940.HomeZAxisAsync` 在这点上更规范，因为它明确使用：
  - `GetActiveLevel(port)`
  - `GetInactiveLevel(port)`

---

## 8.2 MotionAdt8940 的端口极性表达更清晰

`ProcessManager` 用：

- `check_XSTOP0`
- `check_YSTOP0`
- `check_ZSTOP0`

再换算成 `a = 0/1`

`MotionAdt8940` 则直接用：

- `PortItem.IsLowLevelActive`

然后统一通过：

- `GetActiveLevel`
- `GetInactiveLevel`

转换电平。

后者更容易维护，也更不容易在 Z 轴逻辑上出现“变量算了但没真正用对”的问题。

---

## 8.3 MotionAdt8940 目前不支持 Z 光栅端口配置

`HomingOptions` 没有：

- `ZGratingPort`

因此即使以后希望 Z 轴也做光栅回零，当前结构仍需扩展：

- 增加 `ZGratingPort`
- 增加 `HomeZByGratingIoAsync` / `HomeZByLatchAsync` 或统一抽象
- 重新定义 Z 轴回零流程

---

## 9. 最终判断

## 9.1 关于“MotionAdt8940 没有端口参数”的判断

这个判断**不准确**。

更准确的结论是：

- `MotionAdt8940` 的端口参数**不是写在方法参数里**
- 而是封装在 `_homingOptions` 中
- XY 机械端口、XY 光栅端口、Z 机械端口、端口有效电平、速度、超时等都已支持

---

## 9.2 关于“HomeZAxis 应该有 ioMode/latchMode/encoderHomeSuccess”的判断

从当前代码事实看，这个判断**也不成立**。

因为：

- `ProcessManager.HomeZAxis` 本身就没有 Z 光栅尺回零逻辑
- `MotionAdt8940.HomeZAxisAsync` 也没有 Z 光栅尺回零逻辑
- 两边当前设计都只让 XY 支持光栅尺回零

所以现在真正的差异不是：

- `ProcessManager` 有，`MotionAdt8940` 没有

而是：

- **XY：两边都有，只是参数表达形式不同**
- **Z：两边都没有**

---

## 10. 建议

如果你的目标是让 `MotionAdt8940` 完全承接旧 `ProcessManager` 的“回零语义”，建议分两种情况处理：

### 情况 A：只是确认现有 XY 能否覆盖旧逻辑
答案：**基本可以覆盖**。

因为 `MotionAdt8940` 已具备：

- 机械回零
- 光栅 IO 回零
- 光栅锁存回零
- 超时
- 回退脉冲
- Z 回零后抬升
- 端口极性配置

### 情况 B：你希望 Z 轴也支持光栅尺回零
答案：**当前两份代码都还没有完整实现，需要新增设计和实现。**

至少需要补充：

1. `HomingOptions` 增加 `ZGratingPort`
2. 增加 Z 轴光栅 IO / 锁存回零流程
3. 明确 Z 轴是先机械回零再光栅精校，还是直接按光栅回零
4. 在配置模型和 UI 中同步增加对应参数

---

## 11. 一句话总结

**MotionAdt8940 并不是没有回零端口参数，而是把它们封装进 `HomingOptions` 了；XY 光栅尺回零两边都有，Z 光栅尺回零两边其实都没有实现。**
