# 旧字段到新模型映射清单

## 说明

本清单用于补充《旧方案与现有方案对比分析报告》，重点针对：

- `Fredy\Windows\ParamConfig\old\ProcessParameters.cs`
- `Fredy\Windows\ParamConfig\old\ConfigManager.cs`

中的旧字段/旧概念，映射到现有方案中的：

- `Common\Models\Recipe.cs`
- `Fredy\Windows\ParamConfig\ConfigModels.cs`
- `Fredy\ViewModels\NinePointCalibrationViewModel.cs`
- 以及相关运行时服务

状态说明：

- **已迁移**：已有较明确的新字段或新模型承接
- **部分迁移**：能力有承接，但不是一一对应，或只承接了一部分语义
- **未迁移**：当前未发现明确承接位置
- **建议废弃**：旧字段语义已不适合新架构，建议不要直接迁移

## 三、旧字段与现有全局配置 `AppConfig` 的对应关系

以下不是一一字段同名映射，而是“旧语义在现有全局配置中的承接位置”。

| 旧语义 | 现有承接字段 | 状态 | 说明 |
|---|---|---|---|
| 运动控制器连接配置 | `AppConfig.MotionController` | 新增增强 | 旧 `ProcessParameters` 不负责这部分，现有方案新增并独立出来，结构更合理。 |

| 头道特殊工艺默认值 | `AppConfig.FirstPass` | 部分承接 | 适合承接“全局默认值”，但不适合替代配方专属参数。 |

## 五、`old.ConfigManager` 旧键值模式到现有模型的映射建议

`ConfigManager` 本身不是字段模型，而是旧的读写机制。它的“映射”重点不在字段名，而在配置组织方式的迁移。

| 旧机制 | 现有机制 | 状态 | 建议 |
|---|---|---|---|
| `Dictionary<string, object>` 任意存储 | `AppConfig` 强类型模型 | 已迁移 | 不建议继续使用旧字典机制。 |
| 点路径字符串访问嵌套项 | 属性访问 | 已迁移 | 现有方式更适合重构和维护。 |
| 运行时 `JsonElement` 转换 | 编译期模型约束 | 已迁移 | 现有方案更安全。 |
| 单例全局直接存取 | `ConfigService` 统一加载/克隆/保存 | 已迁移 | 保留当前方案。 |
| 自定义配置路径管理 | `ConfigService + PathManager` | 已迁移 | 保留当前方案。 |

### 兼容建议
如果后续需要读取历史配置：

- 只做一次性迁移导入
- 不要让新代码继续直接依赖 `ConfigManager`
- 可编写 `LegacyConfigImporter`，将旧 JSON 转换为：
  - `AppConfig`
  - `RecipeProcessParameters`
  - 未来新增的 `NeedleCalibrationConfig`
  - 未来新增的 `WorkpieceReferenceConfig`

## 七、迁移优先级建议

### 第一优先级：确认是否真的还需要旧“校针参数”
如果九点标定已完全替代旧校针逻辑，则：
- 可以只保留新标定结果
- 旧校针字段标记为“废弃”

如果没有完全替代，则：
- 补 `NeedleCalibrationConfig`

### 第二优先级：确认“工件基准参数”是否仍在业务中使用
如果还需要：
- 新增 `WorkpieceReferenceConfig`
- 配套新增定位/基准服务

如果不再使用：
- 在清单中标记“已废弃”
- 后续不再迁移

### 第三优先级：清理过渡配置
建议后续继续核对：
- `AppConfig.XyDrive`
- `AppConfig.ZAxisBase`

这两组字段与新 `XAxis/YAxis/ZAxis`、以及工艺参数存在一定重叠，后续可以进一步收敛。

## 八、最终结论

从字段层面看：

### 已较完整迁移的部分
- 工件类型
- 头道冲孔参数
- 二道冲孔参数
- 二道检测开关

### 部分迁移的部分
- 当前工艺类型
- 校针偏移语义
- 标定相关流程能力

### 尚未明确迁移的部分
- 校针参考点/状态
- 工件基准坐标/容差/自动校准

### 建议处理原则
- **产品/孔位/冲孔深度相关** -> 放配方 `RecipeProcessParameters`
- **设备标定相关** -> 单独配置模型
- **回零/探测/流程默认值相关** -> 放 `AppConfig` 或专用服务配置
- **流程当前状态** -> 放业务服务/状态机，不放配置文件
