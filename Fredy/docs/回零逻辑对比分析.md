# 回零逻辑对比分析：ProcessManager.cs vs MotionAdt8940.cs

> 生成时间：2025-07  
> 涉及文件：
> - `Fredy/Windows/ParamConfig/old/ProcessManager.cs`（老代码）
> - `HAL/Motion/MotionAdt8940.cs`（新代码）

---

## 一、ProcessManager.cs（老代码）

### 1.1 总体架构

| 特征 | 说明 |
|------|------|
| 执行方式 | 同步阻塞，全部使用 `Thread.Sleep` + `while(true)` 轮询，**会卡 UI 线程** |
| 实例管理 | 手工单例 `ProcessManager.Instance`，无依赖注入 |
| 控制卡耦合 | 方法内部直接 `new CtrlCard()`，强依赖 `visonCAM.GlobalParams.Instance` 全局状态 |

---

### 1.2 Z 轴回零流程（`HomeZAxis`）

```
读取 ZSTOP0 端口 + check_ZSTOP0 电平参数
        │
        ▼
读取当前 IO 电平
        │
   ┌────┴────┐
电平=1       电平=0
   │             │
向负方向      向正方向
连续运动      连续运动
   │             │
阻塞等待电平   阻塞等待电平
变为 1 后停止  变为 1 后停止
   └────┬────┘
        │
   Setup_Pos(3,0,0) 清零坐标
        │
   HomeZLocation > 0 ?
   是 → Axis_Pmove 抬起 + Sleep(1000)
   否 → 跳过
        │
   FlaghomeZ = true
```

> ⚠️ **已知 Bug**：`check_ZSTOP0` 计算出的 `a` 参数从未用于等待条件判断；两个分支
> 均无差别地等待"电平变为 1"，方向与目标电平未正确区分。

---

### 1.3 XY 轴回零流程（`HomeXYAxis`）

```
最多重试 3 次
        │
读取 XSTOP0/YSTOP0 端口 + checkbox 参数，计算 a
        │
检查急停标志（FlagEmergencyStop）
        │
   ┌────┴────────────────────┐
当前电平 == a（触发态）      当前电平 ≠ a（非触发态）
   │                              │
向负方向连续运动              向正方向连续运动
等待电平变 0（超时 10s）      等待电平变 1（超时 10s）
   │                              │
停止 → 正方向移 200 脉冲      停止 → 正方向移 200 脉冲
   │                              │
慢速(100/500/1000)向负方向    慢速向负方向再次找零
等待电平变 0（超时 10s）
   └────────────┬───────────┘
                │
         Setup_Pos 清零坐标
                │
         FlaghomeX/Y = true
```

**急停处理**：停所有轴 + 清全部 `Flaghome*` + 禁用锁存（`Setup_LockPosition`）

## 二、MotionAdt8940.cs（新代码）

### 2.1 总体架构

| 特征 | 说明 |
|------|------|
| 执行方式 | 全 `async/await`，非阻塞，支持 `CancellationToken` |
| 实例管理 | 构造函数依赖注入（`ILogger`、`HomingOptions`） |
| 参数封装 | `HomingOptions` record 集中管理所有回零参数，`HomingPort` 封装端口+有效电平 |
| 调用记录 | `NativeCallRecord` 环形缓冲，可追溯最近 500 条底层调用 |

### 2.2 核心配置结构

```csharp
// 端口描述
HomingPort(int PortIndex, bool IsLowLevelActive)

// 回零参数
HomingOptions(
    HomeSearchSpeed,        // 粗找零速度
    IsIoHome,               // 是否启用 IO 光栅回零
    IsLatch,                // 是否启用硬件 Latch 回零
    IsGratingHome,          // 是否启用光栅回零
    XLimitPort, YLimitPort, ZLimitPort,   // 机械零位端口
    XGratingPort, YGratingPort,           // 光栅端口
    HomeTimeoutMs,          // 统一超时（默认 10000ms）
    HomeBackoffPulse,       // 退让脉冲数（默认 200）
    ZHomeLiftPulse,         // Z 轴抬起脉冲（默认 0）
    ZHomeTowardPositiveDirection,         // Z 轴回零方向
    SlowHomeStartSpeed,     // 慢速起始速度
    SlowHomeSpeed,          // 慢速运行速度
    SlowHomeAcceleration,   // 慢速加速度
    GratingHomeStartSpeed,  // 光栅找零起始速度
    GratingHomeSpeed,       // 光栅找零速度
    GratingHomeAcceleration // 光栅找零加速度
)
```

---

### 2.3 Z 轴回零流程（`HomeZAxisAsync`）

```
读取 ZLimitPort 端口，由 IsLowLevelActive 计算 activeLevel / inactiveLevel
        │
读取当前电平
        │
已在激活态？
   是 → 向反方向运动（ZHomeTowardPositiveDirection 决定），
        等待离开激活态（超时受 HomeTimeoutMs 控制）
        │
向目标方向运动（ZHomeTowardPositiveDirection 决定）
等待到达激活态
        │
ResetPosition 清零坐标
        │
ZHomeLiftPulse > 0 ?
   是 → MoveRelativePulseAsync 异步抬起，等待到位
   否 → 跳过
        │
RecordNativeCall 记录完成
```

---

### 2.4 XY 轴回零流程（`HomeXYAxisAsync`）

```
最多重试 3 次
        │
   ┌────┴──────────────────────────────────────────────────┐
   │  HomeXYMechanicalAsync（机械回零）                     │
   │      │                                                │
   │  当前处于激活态？                                      │
   │  是 → 正方向离开激活态 + HomeBackoffPulse 退缩脉冲     │
   │  否 → 负方向找到非激活沿                               │
   │      │                                                │
   │  慢速（SlowHomeSpeed）正方向找激活沿                   │
   └────┬──────────────────────────────────────────────────┘
        │
  IsGratingHome = true ?
        │
   ┌────┴─────────────────┐
IsLatch                IsIoHome
   │                       │
HomeXYByLatchAsync    HomeXYByGratingIoAsync
（硬件锁存精密回零）  （IO 轮询光栅传感器）
   └────────┬─────────────┘
            │
      ResetPosition 清零坐标
            │
   OperationCanceledException → EmergencyStopAsync → 重新抛出
   Exception → StopAsync → 重试
```

#### Latch 回零子流程（`HomeXYByLatchAsync`）

```
清除锁存状态 → 设置锁存模式（mode=1）
        │
以 GratingHomeSpeed 向负方向连续运动
        │
轮询等待硬件锁存信号（超时 HomeTimeoutMs）
        │
停止轴 → 读取锁存脉冲位置
        │
移动到锁存脉冲位置
        │
finally: 清除锁存模式 + 清除锁存状态
```

## 三、异同对比总结

| 维度 | ProcessManager（老） | MotionAdt8940（新） |
|------|---------------------|---------------------|
| **执行方式** | 同步阻塞，卡 UI 线程 | 全异步，不阻塞 |
| **取消支持** | 手动检查 `FlagEmergencyStop` 全局标志 | `CancellationToken` 标准取消机制 |
| **超时处理** | XY 有 10s 超时；**Z 轴无超时保护** | 统一 `HomeTimeoutMs`，全轴受控 |
| **Z 轴方向** | 硬编码方向逻辑，且有 bug | `ZHomeTowardPositiveDirection` 可配置，逻辑正确 |
| **电平有效性** | `check_XSTOP0` 等 checkbox（Z 轴 `a` 未参与等待判断，存在 bug） | `IsLowLevelActive` 统一封装，无 bug |
| **慢速参数** | XY 硬编码 `startv=100 / speed=500 / acc=1000` | `SlowHomeStartSpeed/Speed/Acceleration` 可配置 |
| **退让脉冲** | XY 硬编码 200 脉冲 | `HomeBackoffPulse` 可配置 |
| **抬起高度** | Z 轴 `HomeZLocation`（脉冲），`Sleep(1000)` 盲等 | `ZHomeLiftPulse` 可配置，异步等待到位 |
| **光栅/Latch** | 不支持 | 支持 IO 光栅 + 硬件 Latch 两种精密回零 |
| **重试机制** | XY 最多 3 次，`continue` 跳转 | XY 最多 3 次，`catch` 后 `StopAsync` |
| **全局状态** | 回零后设置 `FlaghomeX/Y/Z = true` | 无全局标志，调用方通过异常/完成感知结果 |
| **急停处理** | 停全轴 + 清全部标志 + 禁锁存 | `EmergencyStopAsync`，取消即触发，无标志位副作用 |
| **底层调用记录** | 无 | `NativeCallRecord` 环形缓冲，可追溯 500 条 |

---

## 四、核心结论

新代码（`MotionAdt8940.cs`）对老代码（`ProcessManager.cs`）进行了以下层面的全面重构：

1. **正确性**：修复了老代码 Z 轴电平判断 bug（`a` 参数从未生效）。
2. **参数化**：消除全部硬编码数值（退让脉冲、慢速参数、超时、抬起高度、回零方向均可配置）。
3. **异步化**：消除 `Thread.Sleep` 阻塞，改用 `async/await + CancellationToken`，UI 不再卡顿。
4. **可扩展性**：增加光栅 IO 回零与硬件 Latch 精密回零两种高精度模式。
5. **可观测性**：引入 `NativeCallRecord` 调用链追踪，便于问题定位。
6. **解耦**：彻底去除对全局单例 `GlobalParams` 的依赖，改为构造注入。
