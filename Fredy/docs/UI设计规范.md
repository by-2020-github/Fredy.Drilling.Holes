# UI设计规范

## 1. 设计目标

本项目界面采用**简洁、克制、清晰的工业风格**，强调信息准确传达、操作高效、状态明确，避免多余装饰和复杂视觉干扰。

设计原则如下：

- **白色背景**：整体界面背景以白色为主，保持干净、明亮、易读。
- **黑色字体**：前景文字统一使用黑色或高对比深色，确保信息清晰可辨。
- **简洁工业风**：视觉表达偏功能化、理性化，弱化花哨效果，突出结构、层级与状态。
- **高可读性**：适用于设备控制、参数查看、状态监控等工业操作场景。
- **一致性**：颜色、间距、控件、交互反馈保持统一。

## 2. 整体视觉风格

### 2.1 风格定位

- 风格关键词：**简洁、直接、稳定、专业、工业化**。
- 避免使用过多渐变、阴影、玻璃拟态、拟物化装饰。
- 优先通过排版、留白、边框、分组来表达界面层级。
- 所有设计应以“**便于快速识别与操作**”为第一优先级。

### 2.2 页面气质

- 界面应给人**整洁、可靠、可控**的感受。
- 常用页面应优先突出：当前状态、关键参数、主要操作按钮、异常提示。
- 非核心信息弱化显示，但仍需保持清晰可读。

## 3. 色彩规范

### 3.1 基础色

| 类型 | 建议颜色 | 用途 |
|---|---|---|
| 主背景色 | `#FFFFFF` | 页面主背景、面板背景 |
| 主文字色 | `#000000` | 标题、正文、主要数据 |
| 次级文字色 | `#333333` | 次要说明、辅助文本 |
| 分割线/边框色 | `#D9D9D9` | 区块边界、输入框边框、表格分隔 |
| 弱背景色 | `#F5F5F5` | 次级区域、工具栏、只读信息块 |

### 3.2 功能色

功能色用于状态表达，应少量、明确、统一：

| 类型 | 建议颜色 | 用途 |
|---|---|---|
| 成功/运行正常 | `#2E7D32` | 成功提示、设备正常、完成状态 |
| 警告 | `#ED6C02` | 参数临界、需关注状态 |
| 错误/报警 | `#D32F2F` | 故障、异常、危险操作提示 |
| 信息/选中 | `#1976D2` | 当前选中、信息提示、可交互强调 |

状态标识允许使用**红、黄、蓝、绿**四色进行统一表达，推荐映射如下：

| 颜色 | 建议语义 | 示例状态 |
|---|---|---|
| 红 | 报警 / 故障 / 危险 | 故障、急停、异常报警 |
| 黄 | 警告 / 待处理 / 注意 | 预警、待机、需人工确认 |
| 蓝 | 运行中 / 信息 / 处理中 | 运行中、连接中、信息提示 |
| 绿 | 正常 / 完成 / 可用 | 正常、已完成、设备就绪 |

### 3.3 使用要求

- 默认界面以**黑、白、灰**为主，不得大面积使用高饱和颜色。
- 功能色仅用于表达状态、反馈和关键交互，不作为大面积装饰。
- 文本与背景必须保持足够对比度，确保远距离和长时间查看时仍具备良好可读性。
- 同一类状态在全系统中应保持固定颜色，不得在不同页面随意变更语义。

## 4. 字体规范

### 4.1 字体要求

- 默认使用系统无衬线字体。
- 中文与数字应清晰、稳定、易读。
- 避免使用花哨字体、过细字体或装饰性字体。

### 4.2 字号层级

建议层级如下：

| 层级 | 建议字号 | 字重 | 用途 |
|---|---|---|---|
| 页面标题 | 20-24px | SemiBold/Bold | 页面名称、总览标题 |
| 区块标题 | 16-18px | SemiBold | 模块标题、分组标题 |
| 正文/表单 | 13-14px | Regular | 常规文本、输入标签 |
| 辅助信息 | 12px | Regular | 说明、注释、时间、次级信息 |
| 关键数据 | 18-24px | Bold | 状态值、坐标值、产量等关键数字 |

### 4.3 文字规则

- 文字颜色默认使用黑色或深灰色。
- 同一页面不要出现过多字号。
- 标题、正文、说明文字层级必须明确。
- 参数名称与参数值应对齐显示，便于快速扫描。

## 5. 布局规范

### 5.1 总体布局

- 页面结构应清晰分区，推荐采用：**顶部信息区 + 主操作区 + 状态区/日志区**。
- 内容布局应规整对齐，优先采用网格化或分栏式布局。
- 每个功能区域需有明确边界，可通过标题、边框、留白进行区分。

### 5.2 间距规范

建议采用统一间距体系：

- 最小间距：4px
- 紧凑间距：8px
- 常规间距：12px
- 模块间距：16px
- 大区块间距：24px

要求：

- 同类控件之间保持一致间距。
- 页面边缘与内容之间应保留足够留白。
- 不允许控件排列杂乱、对齐线不统一。

### 5.3 对齐原则

- 文本左对齐优先。
- 数值信息建议右对齐或按位对齐，便于比较。
- 标签与输入框、按钮与按钮之间保持统一基线。
- 表单、面板、表格尽量使用统一宽度和统一节奏。

## 6. 控件规范

### 6.1 按钮

- 主按钮用于主要操作，如启动、保存、确认。
- 次按钮用于辅助操作，如重置、返回、取消。
- 危险操作按钮应明确使用警示色，并增加确认机制。
- 按钮文案应直接明确，优先使用动词，如“启动”“停止”“保存参数”。

按钮要求：

- 尺寸统一，避免同页面大小不一。
- 状态明确区分：默认、悬停、按下、禁用。
- 禁止使用过重阴影和复杂立体效果。

### 6.2 输入框

- 输入框边框清晰，背景简洁。
- 必填项、只读项、异常项应有明显区分。
- 数值输入应标注单位。
- 参数输入区域需尽量靠近对应说明，减少误操作。

### 6.3 表格

- 表格用于展示参数、记录、状态列表。
- 表头需清晰，文字简洁。
- 行高适中，保证扫描效率。
- 重点列可适度加粗或用深色强调，但不得破坏整体简洁性。

### 6.4 卡片/面板

- 用于承载独立模块，如设备状态、坐标信息、任务信息。
- 面板背景保持白色或浅灰色，搭配浅边框区分层级。
- 每个面板只承载一组相关信息，避免内容堆叠过多。

## 7. 状态与反馈规范

### 7.1 状态表达

设备类系统应明确区分以下状态：

- 未连接
- 待机
- 运行中
- 暂停
- 报警
- 故障
- 完成

要求：

- 状态文本必须清晰可读，不仅依赖颜色区分。
- 关键状态建议同时使用颜色 + 文本 + 图标。
- 报警和故障信息必须突出，但不能造成界面混乱。
- 推荐统一使用：**红=报警/故障，黄=警告/待处理，蓝=运行/信息，绿=正常/完成**。

### 7.2 提示信息

- 成功提示简洁明确，不干扰主流程。
- 警告提示需说明原因及建议处理方式。
- 错误提示需包含问题描述，必要时提供可执行动作。
- 弹窗仅用于关键确认、风险操作、重要异常。

## 8. 数据展示规范

- 核心数据优先展示，如设备状态、加工数量、坐标值、报警信息。
- 数值显示需统一格式，如小数位、单位、时间格式。
- 实时数据区域应强调稳定刷新和清晰识别，避免频繁闪烁。
- 监控信息应做到“**一眼可读**”，减少理解成本。

## 9. 图标与图形规范

- 图标风格应简洁、线性、统一。
- 图标仅用于辅助识别，不替代文字说明。
- 不使用过于花哨、卡通化或娱乐化图标。
- 状态图标应优先服务于工业设备语义，如运行、停止、报警、连接、断开。

## 10. 禁止项

为保持统一工业风格，以下设计应尽量避免：

- 大面积高饱和颜色背景
- 复杂渐变、强阴影、发光效果
- 过圆润或娱乐化的控件造型
- 花哨动画和无意义动效
- 一页中出现过多颜色、字号、边框样式
- 难以辨认的浅灰文字或低对比度组合

## 11. 推荐总结

本项目 UI 应保持以下核心特征：

- **背景白色**
- **前景字体黑色**
- **黑白灰为主，功能色少量点缀**
- **布局规整，信息分区明确**
- **弱装饰、强信息、强可读性**
- **整体呈现简洁工业风格**

在具体设计与实现时，应始终以**清晰、稳定、高效、易操作**为第一原则。
